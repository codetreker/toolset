# 项目验收（Project Acceptance）

## 铁律

- **e2e = 浏览器 + 截图**，不是 HTTP 状态码
- 所有带 UX 的验收**必须**浏览器打开、截图确认
- `curl` 看状态码 ≠ e2e 验收
- UX 验收由 QA（烈马）执行

## 验收流程

1. **飞马审批 PR** → 在项目频道通知 QA 验收，附上：PR 链接、部署地址、验收标准
2. **QA 按清单逐项检查**（见下方模板）
3. **UX 项：浏览器打开 → 截图为证**（用 `playwright-browser` skill 或手动浏览器）
4. **结果通报**：
   - ✅ 通过 → 频道确认，附截图
   - ❌ 不通过 → 列出具体问题（哪项、什么现象、截图），打回开发
5. **全部通过** → 任务板标记 Done

## 验收类型

### 1. 功能验收

验证 API 端点、路由、数据正确性。

- 用 `curl` / `httpie` 测试 API 响应和数据结构
- 用 `npm test` 跑测试套件（单测 + 集成测试）
- 验证正常路径、边界条件、错误路径

### 2. UX 验收（必须浏览器 + 截图）

验证页面渲染、交互、响应式布局。

- **桌面端**：浏览器打开目标页面，截图确认
- **移动端**：用 Playwright 模拟移动设备视口，截图确认
- 检查项：页面加载、导航、表单交互、错误提示、空状态
- 截图命名：`acceptance-<功能>-<设备>-<日期>.png`

```bash
# Playwright 截图示例（桌面）
npx playwright screenshot --wait-for-timeout=3000 "https://目标地址" desktop.png

# Playwright 截图示例（移动端模拟）
npx playwright screenshot --device="iPhone 14" --wait-for-timeout=3000 "https://目标地址" mobile.png
```

> 也可以用团队的 `playwright-browser` skill 中的脚本。

### 3. 安全验收

- 路径穿越：尝试 `../` 路径访问受限资源
- 注入：SQL 注入、命令注入测试
- XSS：在输入字段注入 `<script>alert(1)</script>`，浏览器打开确认是否转义
- 认证/授权：未登录访问受保护路由，越权访问他人资源

### 4. 性能验收

- 响应时间：关键 API < 预期阈值
- 大数据量：导入/查询大量数据，观察是否超时或崩溃
- 页面加载：浏览器打开，观察首屏加载时间

## 验收清单模板

每次验收时复制此模板到频道或笔记中，逐项打勾：

```markdown
## 验收清单 - [任务名称] - [日期]

**PR**: #xxx
**部署地址**: https://...
**验收人**: 烈马

### 功能验收
- [ ] 测试套件通过（npm test）
- [ ] API 端点响应正确（正常路径）
- [ ] API 边界条件和错误路径
- [ ] 数据持久化/查询正确

### UX 验收（浏览器 + 截图）
- [ ] 桌面端页面正常渲染（截图: ）
- [ ] 移动端页面正常渲染（截图: ）
- [ ] 核心交互流程正常（截图: ）
- [ ] 错误状态/空状态显示正确（截图: ）
- [ ] 导航和路由正确

### 安全验收
- [ ] 路径穿越测试
- [ ] 输入注入测试
- [ ] XSS 测试（浏览器确认）
- [ ] 认证/授权边界

### 性能验收
- [ ] 关键 API 响应时间达标
- [ ] 大数据量场景无超时/崩溃
- [ ] 页面首屏加载可接受

### 结论
- [ ] ✅ 全部通过 → Done
- [ ] ❌ 不通过 → 问题列表：
```

## 工具速查

| 用途 | 工具 |
|------|------|
| 浏览器截图 | `playwright-browser` skill / `npx playwright screenshot` |
| API 测试 | `curl` / `httpie` |
| 测试套件 | `npm test` |
| 移动端模拟 | Playwright `--device` 参数 |

## 注意事项

- 截图必须附在验收报告中，不能只说"看了没问题"
- 安全验收中的 XSS 测试也要浏览器打开确认，不能只看响应体
- 验收不通过时，问题描述要具体到可复现（步骤 + 截图 + 期望 vs 实际）
- 不同项目可按需裁剪清单，但 UX 截图是硬性要求
